Integrating into Elliptic

A guided walkthrough of the decisions a new customer makes when wiring Elliptic into their compliance stack — from how rules screen activity, to investigations, data access, and reporting.

⚡
How to read this
An Elliptic integration is a small set of choices across four areas. Most of it is API-led, and CoPilot (AI assistance) is optional at every step. Make selections below — your choices assemble into an integration blueprint at the bottom.
Stage 01

Risk rule configuration

Decide how Elliptic screens activity and what should create an alert. This is where proximity tuning, monitoring strategy, and AI assistance are set.

A · Proximity-based rule tuning

Choose how close exposure has to be before it creates an alert. Pick a rule type to see how it behaves and how it surfaces through the API.

B · Monitoring strategy

Keep prior screening decisions current. Pick how Elliptic re-evaluates activity when new intelligence changes the risk picture — and whether CoPilot summarizes the change.

C · Webhook alerts — threshold

Fire a webhook the moment a screening result crosses a risk threshold, so flagged activity pushes into your systems in real time — no polling. Set the score that should trigger it.
